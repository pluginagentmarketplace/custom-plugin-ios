# /learn

Start your personalized learning journey through the developer roadmap ecosystem.

This command helps you discover the right learning path based on your goals and current experience level.

## Usage

```
/learn
```

## What This Does

When you use this command, I'll:

1. **Assess your goals**: What do you want to build or learn?
2. **Understand your background**: What experience do you have?
3. **Recommend a path**: Suggest the best learning sequence
4. **Set milestones**: Break down learning into manageable chunks
5. **Suggest resources**: Point you to specific roadmaps and skills

## Example Scenarios

### Scenario 1: Complete Beginner
**Goal**: "I want to become a web developer"
**Response**:
- Foundational path (6 months): HTML ‚Üí CSS ‚Üí JavaScript ‚Üí React
- Recommended: Frontend & Web Technologies expert
- Skills to focus on: frontend-frameworks
- Milestones: Build 5 projects, progressively complex

### Scenario 2: Backend Developer Wanting AI
**Goal**: "I know Python and backend, want to learn ML"
**Response**:
- Accelerated path (3-4 months): AI Foundations ‚Üí Machine Learning Practice
- Recommended: AI & ML Specialist
- Prerequisites met: Python, problem-solving
- Projects: Build 3 ML models, deploy one

### Scenario 3: Career Switcher
**Goal**: "I'm transitioning from sales to engineering"
**Response**:
- Foundation first: Fundamentals (Git, Linux, CS basics)
- Then choose specialization (frontend, backend, etc.)
- Recommended timeline: 6-12 months
- Milestone-based progression

### Scenario 4: Senior Engineer Wanting Architecture
**Goal**: "I want to move into system design and architecture"
**Response**:
- Already have: Most technical fundamentals
- Focus: Specialized Roles & Architecture expert
- Deep dive: System design patterns, distributed systems
- Career milestone: Principal/Staff engineer
